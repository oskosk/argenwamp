Descripción

ArgenWAMP es una herramienta que te permite construir mapas web con datos guardados en una Google Spreadsheet. El código de argenWAMP está disponible bajo una licencia libre.

  • Este mapa va a estar siempre online tengas o no servidor porque se basa en la infraestructura de Google Docs.
  • Podés incrustar el mapa en tu web o compartir el link.

Puede traer datos de múltiples fuentes y soporta:

  • Marcadores a partir de coordenadas o Geocoding (basado en nominatim de OpenStreetMap).
  • Capas KML ya publicadas en la web.
  • Capas WMS en SRS=EPSG:3857.

Cómo funciona ArgenWAMP

  • ArgenWamp es altamente disponible porque está hosteado en GitHub Pages.
  • El código de argenWAMP está en la branch master del repo http://github.com/oskosk/argenwamp.
  • La versión live, osea el sitio argenwamp.com está en la branch gh-pages del mismo repo y ahí es donde aparece la magia de GitHub y mantiene esa página siempre online por el resto de la eternidad.
  • La estabilidad del mapa depende de la estabilidad de la librería argenmap.jquery, que es un servicio WEB del IGN.

Estructura de ArgenWAMP

ArgenWAMP tiene 3 partes y usa unas librerías hosteadas en cdnjs.com. Eventualmente, las voy a ofrecer minificadas. Las partes son:

  • Un index.html que es lo que estás viendo ahora
  • Un poco de javascript con forma de plugin de jQuery. El plugin está hecho siguiendo los patrones de diseño de jQuery Boilerplate así que es bastante reusable. Internamente, lo nombré Argenmap Vis y por eso el archivo se llama argenmapvis.jquery.js.
  • Un iframe.html que es el html que arma el mapa en pantalla completa que sirve para ser incrustado en un blog o cualquier web usando la etiqueta HTML iframe.

Criteria

  • Muchas personas están acostumbradas a las planillas de cálculo como estructuras de datos.
  • El código de esta herramienta consume JSON.
  • Google ofrece por cada documento publicado una url con el documento JSON que representa al documento spreadsheet. Es por eso que esta herramienta sirve para consumir JSON de otra fuente.
  • Sí, los "datos" quedan guardados en google. Pero, no hace falta guardar los datos más importantes del mundo en este tipo de mapas.
  • Es una herramienta útil para bocetar mapas a partir de KML o servicios WMS (datos ya publicados conforme a estándares abiertos).
  • Este proyecto se propone acercar los servicios WMS y los KML ya publicados a quienes no dispongan de mayores habilidades informáticas que el manejo de una planilla de cálculos y una cuenta de Google.
  • Esta herramienta está construída como 2 html y una plugin de jquery. que interpreta json. Sea cual sea el origen (Google Docs, una URL de un documento JSON o un objeto javascript).

Cómo se usa argenWAMP

ArgenWAMP permite definir un mapa completando celdas en la hoja de cálculo.

Esta es la descripción acerca de cómo completar las columnas de la hoja de cálculo para generar un mapa.

Columna Tipo_recurso

Los valores posibles son:

centro
define a esta fila como un descriptor de la vista inicial del mapa.
kml
define a esta fila como un descriptor de una URL de un KML.
marcador
define que a esta fila va describir un punto geográfico sobre el mapa.
wms
define a esta fila como un descriptor de una capa de datos en un servicio de mapas WMS.
cluster_de_marcadores
Esta fila habilita el clustering de marcadores. En caso de tener muchos marcadores, esta función permite que se muestren agrupaciones de marcadores resumidas en un ícono que indica la cantidad de marcadores en la zona resumida. Para deshabilitar esta funcionalidad, simplemente borrá esta fila y se mostrarán todos los marcadores sin ser agrupados.

Columna Titulo

Este campo sólo sirve para las filas del tipo marcador y centro.

En la fila de tipo centro
Define el título del mapa. Esto aparece sobre un rectángulo translucido.
En las filas de tipo marcador
Define la etiqueta de un marcador, que se ve al ponel el mouse soebre el marcador sin clickear y el título de que aparecerá en el globito que presenta la información de la columna Descripcion al hacer click sobre el marcador.

Columna Recurso

Este campo es la parte importante de cada fila. Acepta un par de coordenadas o un texto que argenWAMP intentará geocodificar.

En la fila de tipo centro
Define el centro inicial del mapa.
En las filas de tipo marcador
Define la posición geográfica del punto que el marcador representa.
En las filas de tipo KML
Define la URL del KML que queremos superponer sobre el mapa.
En las filas de tipo WMS
Define la URL base del servicios WMS. No la URL del capabilities.

Columna Capa

El nombre de esta columna sólo tiene sentido para las filas de tipo WMS y marcador.

En la fila de tipo WMS
Define qué capas queremos del servicios WMS. Este texto se pasa directamente como parámetroLAYERS al servicio WMS, es decir podés especificar más de una capa separa por comas, sin dejar espacios.
En las filas de tipo marcador
Define la la URL de un ícono para utilizar como marcador. Debería ser un png que esté online en culquier URL. No hace falta que vos publiques el ícono si ya tenés la URL de un PNG que esté online.

Columna Grupo

El valor de esta columna sólo tiene sentido para las filas de tipo marcador.

Los marcadores se agruparán en grupos según esta columna. Al cargar, por ejemplo 15 marcadores (filas del tipo marcador), y especificar "hospital" en la columna grupo de 10 de ellos y luego "comisaría" en la columna grupolos otros 5, el mapa mostrará un selector de grupos en el cual se pueden ocultar o mostrar los distintos grupos de marcadores.
